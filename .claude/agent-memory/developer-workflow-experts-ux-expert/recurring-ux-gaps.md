Чек-лист повторяющихся UX-пробелов в спеках DateCountdown. Проверять в каждом ревью спеки/фичи.

**Why:** в MVP-спеке (2026-05-25-datecountdown-mvp.md) эти классы пробелов отсутствовали как AC, а это самые частые источники регрессий UX на Android.

**How to apply:** при ревью любой спеки этого проекта явно искать наличие AC по каждому пункту; отсутствие = находка.

1. **Доступность (a11y) как класс** — нет AC про contentDescription для иконок-эмодзи, роль/состояние выбора в color/icon grid, focus order в sheet, объявление snackbar для TalkBack. Самый крупный системный gap.
2. **fontScale / Dynamic Type** — крупные числа счётчика (до 220px) без AC про clamp/autosize/scroll при system fontScale 1.3+.
3. **Потеря данных в bottom sheet** — ввод в create-режиме при повороте, dismiss-by-drag, process death. Нужен AC: confirm-dialog / draft / явное «теряется».
4. **Неинтерактивный видимый UI** (anti-pattern) — фильтр-чипы и поиск «видны, но no-op». Тап без реакции путает. Выбрать одно решение (скрыть vs disabled-визуал).
5. **Permission denied feedback** — отказ POST_NOTIFICATIONS / недоступность exact alarm: форма обратной связи пользователю (баннер/индикатор/настройки) часто не определена.
6. **Тач-цели <48dp** — макет содержит 36–40dp элементы; спека должна нормировать минимум 48dp.
7. **RTL / landscape / tablet** — если не поддерживаются, должны быть явно в Out of Scope, иначе «свайп влево» и «2-колоночный грид» поломаются при расширении.
8. **Edge формы строк** — «−0 дней назад», очень длинный title на counter, дефолт времени у границы суток.
9. **A11y-конвенция ≠ автоматически контракт** — вынос a11y в сквозную конвенцию (docs/conventions/accessibility.md) НЕ закрывает gap сам по себе. Контрактом она становится только если у каждой фичи есть AC «a11y-набор», перечисляющий конкретные элементы экрана + применённые правила. Тонкие per-feature ссылки (одна-две на правило 3/5) оставляют /acceptance без L5-чеклиста: правила 1 (contentDescription конкретных иконок), 4 (scrim на каких именно hero-палитрах), 6 (snackbar liveRegion + undo для TalkBack), 7 (focus order в sheet), 8 (live-preview спамит TalkBack) остаются неверифицируемыми. Past-counter (03) и notifications (07) легко остаются вовсе без a11y AC.
10. **No-op видимый UI + a11y** — фиксация «тап no-op» (AC-LS-2/3) закрывает п.4, но порождает новый: фокусируемый элемент без действия. Решение «hide vs disabled-visual» должно нести семантику недоступности (disabled state / убрать из focus order), иначе TalkBack объявляет кликабельный элемент, который не реагирует.
