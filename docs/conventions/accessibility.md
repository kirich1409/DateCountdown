# UI Accessibility — конвенция проекта (применяется всегда)

Доступность — **сквозной принцип**, а не отдельная фича. Эти требования обязательны для **любого**
UI-кода в проекте (каждый экран, компонент, состояние). Не «добавляется потом» — закладывается сразу.
Code review и `/finalize` проверяют их наравне со стилем. Конкретные фичи ссылаются сюда вместо
дублирования.

Область: Jetpack Compose + Material 3, Android. Базовый ориентир — Material 3 a11y и Android
accessibility guidelines.

## Обязательные правила

1. **contentDescription для всего, что несёт смысл.** Каждая иконка-кнопка, иконка события, статусная
   иконка имеет осмысленный `contentDescription` (локализованный). Чисто декоративные элементы (blob-фоны
   счётчика, разделители) — `contentDescription = null` явно. Иконки-эмодзи событий озвучиваются названием
   иконки/события, не «кнопка».

2. **Состояние выбора — не только цветом/заливкой.** Выбранные элементы (пикеры цвета/иконки, фильтр-чипы,
   таб) несут неколор-сигнал (галочка, обводка, смена формы) И семантику `selected`/`stateDescription`
   («выбрано»/«не выбрано»). Запрещено полагаться исключительно на цвет.

3. **Тач-цели ≥ 48×48 dp.** Все интерактивные элементы. Если визуальный размер меньше (чипы 36dp,
   ячейки пикеров 40dp) — расширять область касания (`Modifier.minimumInteractiveComponentSize()` /
   паддинг), сохраняя визуал из макета.

4. **Контраст.** Текст ≥ 4.5:1 (≤18sp) и ≥ 3:1 (крупный). Проверять все пары токенов
   `container`/`onContainer`, `hero`/`onHero` (9 событийных палитр × светлая/тёмная) и текст на
   полупрозрачных Glass-элементах поверх светлых hero (amber/orange) — при нехватке добавлять scrim.

5. **Динамический шрифт (fontScale).** Layout не ломается при максимальном системном размере шрифта:
   крупные числа счётчика — autosize/clamp; подписи и заголовки — без обрезки/перекрытия; списки/карточки
   не наезжают. Не фиксировать высоты так, чтобы текст обрезался.

6. **Объявление транзиентного UI screen reader'у.** Snackbar (в т.ч. «Удалено · Отменить»), смена
   состояния (loading→content, ошибка) объявляются (`liveRegion`/assertive). Действие undo достижимо
   screen-reader-пользователем (таймаут не отрезает доступ — продлевать/давать альтернативу).

7. **Порядок фокуса и видимый фокус.** В формах/sheet логический порядок (например: название → дата →
   цвет → иконка → Сохранить); видимый focus-индикатор при клавиатурной/switch-навигации.

8. **Не полагаться на анимацию/таймаут как единственный канал.** Информация, доступная только через
   исчезающий элемент, дублируется доступным способом.

9. **TalkBack-проверка экрана** — часть acceptance для каждого нового экрана (см. testing pyramid L5).

## Как применять
- В per-feature спеках точечные a11y-AC (тач-цели, contentDescription набора иконок) ссылаются на эту
  конвенцию.
- В реализации Compose: `Modifier.semantics { ... }`, `contentDescription`, `stateDescription`,
  `liveRegion`, `minimumInteractiveComponentSize()`.
- Нарушение этих правил = blocker на code review, как и нарушение стиля.
