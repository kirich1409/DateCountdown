# 02 · Логика счётчика

Чистая логика остатка времени (`:domain`, kotlinx-datetime) + её отображение на полноэкранном
счётчике и карточке. Прошедшие — в [03](03-past-events.md).

Эталоны: `../screenshots/counter-near-3d.jpg` (3 дня), `counter-far-248d.jpg` (248 дней),
`counter-today-4h.jpg` (4 часа), `counter-long-9y253d.jpg` (9 лет 253 дня).

## Вычисление остатка (`:domain`, pure)
- **AC-CL-1** Остаток = `targetDateTime − now` в годах/днях/часах/минутах/секундах, через kotlinx-datetime
  с `TimeZone.currentSystemDefault()`. Чистая функция от (`target`, `now`).
- **AC-CL-2** При `target − now <= 0` функция возвращает признак **past** (см. [03](03-past-events.md)); счётчик
  НЕ показывает «0 секунд» и не уходит в отрицательные значения отсчёта. (Эталон past: [03](03-past-events.md).)

## Primary unit (первая ненулевая)
- **AC-CL-3** Primary выбирается сверху вниз: годы → дни → часы → минуты → секунды. Если все нули — секунды.
- **AC-CL-4** primary=годы → отображаются **годы (крупно) и дни (мельче) в столбик**, обе с плюрализацией
  (эталон `counter-long-9y253d.jpg`: «9 лет» над «253 дня»).
- **AC-CL-5** primary=дни → одно крупное число дней (эталон `counter-near-3d.jpg` «3 дня», `counter-far-248d.jpg` «248 дней»).
- **AC-CL-6** primary=часы/минуты/секунды → одно крупное число соответствующей единицы (эталон `counter-today-4h.jpg` «4 часа»).
- **AC-CL-7** «Сегодня» (0 дней, но ненулевые часы/минуты/секунды): primary берётся из часов/минут/секунд по AC-CL-3 (эталон `counter-today-4h.jpg`).

## Glass-ряд (нижние ячейки полноэкранного счётчика)
- **AC-CL-8** Показывает только единицы МЛАДШЕ primary, по таблице:

  | primary | ячейки glass-ряда |
  |---|---|
  | годы | часы, минуты, секунды (3) |
  | дни | часы, минуты, секунды (3) |
  | часы | минуты, секунды (2) |
  | минуты | секунды (1) |
  | секунды | ряд скрыт (0) |

  Эталоны: `counter-near-3d.jpg`/`counter-long-9y253d.jpg` — 3 ячейки; `counter-today-4h.jpg` — 2 ячейки (минут/секунд).
- **AC-CL-9** Значения в ячейках с ведущим нулём (`02`, `08`), подпись единицы капсом под числом.

## Живой тик
- **AC-CL-10** На полноэкранном счётчике значения обновляются раз в секунду, пока экран виден.
- **AC-CL-11** При уходе экрана в фон тик останавливается (нет фоновой работы/рекомпозиции). Источник тика —
  pure `Flow<Instant>` 1Гц в `:domain`; привязка к жизненному циклу — **только в `:feature:counter`** через
  Decompose/Essenty Lifecycle (`doOnStart`/`doOnStop`).
- **AC-CL-12** Карточки списка **никогда не тикают посекундно**: минимальная гранулярность карточки — день
  (показывают число дней, AC-LS-5). Tick-`Flow` в `:feature:list` не используется; посекундной рекомпозиции
  грида нет. (Живой посекундный тик — только на полноэкранном счётчике, AC-CL-10.)

## Заголовок / шапка счётчика (upcoming)
- **AC-CL-13** Топ-бар upcoming-счётчика: `arrow_back`, `edit`, `more_vert`. **Кнопка share отсутствует** в MVP (см. [09](09-edge-cases.md) AC-EDGE-9). `more_vert` всегда содержит пункт «Удалить» (удаляет событие с undo-snackbar, [04](04-list-screen.md) AC-LS-9, → возврат на список).
- **AC-CL-14** Шапка: blob-иконка события, лейбл «ДО СОБЫТИЯ», название события, чип с датой («30 апреля 2026 · 11:00»). Фон — `hero`-цвет события (эталоны).

## Робастность
- **AC-CL-15** При максимальном системном fontScale primary-число и подписи не обрезаются и не перекрывают
  glass-ряд/заголовок (autosize/clamp/допустимый перенос). См. [конвенцию a11y](../../conventions/accessibility.md) (правило 5, fontScale).
- **AC-CL-16** Длинный `title` на счётчике — максимум 2 строки с ellipsis (RU и EN). Большие значения «9 лет 253 дня» — без переполнения (эталон `counter-long-9y253d.jpg`). См. [09](09-edge-cases.md).

## Доступность (экран) — по [конвенции](../../conventions/accessibility.md)
- **AC-CL-17** contentDescription: иконка события (озвучивается названием события/иконки), `arrow_back` («Назад»), `edit` («Редактировать»), `more_vert` («Ещё»). Blob-фигуры фона — декоративны (`contentDescription = null`).
- **AC-CL-18** Контраст: текст и числа на `hero`-фоне проходят ≥4.5:1 (≤18sp) / ≥3:1 (крупное); на светлых hero (amber/orange) числа/подписи и текст Glass-ячеек получают scrim/затемнение при нехватке контраста (правило 4 конвенции).
- **AC-CL-19** Primary-число и подписи озвучиваются осмысленно («3 дня до события»), а не как набор цифр.
