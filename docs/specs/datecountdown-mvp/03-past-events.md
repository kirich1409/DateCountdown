# 03 · Прошедшие события

Прошедшее — производный статус события (`targetDateTime <= now`), не отдельная сущность.
Эталоны: `../screenshots/counter-past.jpg` (past-счётчик), `list-with-past.jpg` (секция в списке).

## Переход в past
- **AC-PE-1** Событие становится past, как только `now >= targetDateTime`, без действий пользователя.
- **AC-PE-2** Для past-события уведомление не запланировано / отменено (см. [07](07-notifications.md) AC-NT-4).
- **AC-PE-3** `daysAgo` = целое число полных дней с `targetDateTime` до `now` (TimeZone.currentSystemDefault()).

## Карточка past (в списке, секция «Прошедшие»)
- **AC-PE-4** Фон — `surfContHigh` (серый); иконка приглушена; бейдж «ПРОШЛО» (`surfCont`).
- **AC-PE-5** Крупное `−N` (N = daysAgo), строка «N дней назад» (плюрализация), название **зачёркнуто**.
  Эталон: `list-with-past.jpg` (карточки «−25», «−43»).

## Past-счётчик (полноэкранный)
Эталон: `counter-past.jpg`.
- **AC-PE-6** Фон — `surfContHighest` (нейтральный graphite, не цвет события); приглушённая blob-иконка.
- **AC-PE-7** Чип «СОБЫТИЕ В ПРОШЛОМ» (`errorContainer`/`onErrorContainer`), название, строка даты.
- **AC-PE-8** Крупное `−N`, подпись «N дней назад» (плюрализация).
- **AC-PE-9** Внизу две кнопки: **Перенести** (`secondaryContainer`, иконка event_repeat) и **Удалить** (`errorContainer`, иконка delete).
- **AC-PE-10** Топ-бар past-счётчика: `arrow_back` + (по дизайну) `more_vert`. Поскольку есть явная кнопка
  «Удалить» и редактирование идёт через «Перенести», **`more_vert` на past-счётчике скрыт** (во избежание
  дублирования «Удалить»; правка ревью UX). Кнопки `edit`/`share` на топ-баре past отсутствуют.
- **AC-PE-11** Если `daysAgo == 0` (событие прошло сегодня) — подпись «Сегодня» (не «0 дней назад»); число `−0` не показывается, вместо крупного числа — «Сегодня».

## Действия
- **AC-PE-12** «Удалить» удаляет событие (с undo-snackbar, см. [04](04-list-screen.md) AC-LS-9) и возвращает на список.
- **AC-PE-13** «Перенести» открывает экран редактирования этого события (предзаполнен) — см. [05](05-add-edit.md) AC-AE-9, [06](06-navigation.md) AC-NAV-6. После сохранения **с будущей датой** событие становится upcoming, для него планируется уведомление; навигация возвращается на счётчик, который становится полноценным upcoming-счётчиком (тик + glass-ряд активны, [06](06-navigation.md) AC-NAV-6).
- **AC-PE-13a** Если при «Перенести» снова выбрана дата в прошлом — событие остаётся past, навигация возвращается на past-счётчик, уведомление не планируется ([07](07-notifications.md) AC-NT-4).

## Список со смешанными событиями
- **AC-PE-14** Прошедшие показываются отдельной секцией ниже upcoming (заголовок «Прошедшие · N», свёртка) — спецификация секции в [04](04-list-screen.md) AC-LS-11..13.
- **AC-PE-15** Сортировка past-карточек — по убыванию `targetDateTime` (недавно прошедшее первым).

## Доступность (экран) — по [конвенции](../../conventions/accessibility.md)
- **AC-PE-16** Крупное `−N` и подписи не обрезаются при максимальном системном fontScale (autosize/clamp; правило 5 конвенции).
- **AC-PE-17** contentDescription: иконка события, `arrow_back` («Назад»). Кнопки «Перенести» и «Удалить» — осмысленные label'ы и тач-цели ≥48dp (правила 1, 3). Чип «Событие в прошлом» озвучивается как статус. `−N` озвучивается как «N дней назад».
