# 04 · Главный список

Эталоны: `../screenshots/list-default.jpg` (8 событий), `list-swipe-delete.jpg` (свайп),
`list-with-past.jpg` (секция «Прошедшие»), `empty-state.jpg` (пусто).

## Шапка и фильтры
- **AC-LS-1** Large Top App Bar: заголовок «Обратный отсчёт», сабтайтл «N событий впереди» (N и форма
  слова склоняются, считаются только upcoming; эталон «8 событий впереди», «4 события впереди»).
- **AC-LS-2** Leading-иконка — `menu` (гамбургер, визуально по дизайну); drawer вне MVP. Trailing — `search` (вне MVP) и `more_vert`. **Неактивные в MVP `menu` и `search` имеют `disabled`-семантику** (не фокусируются screen reader'ом / объявляются недоступными, не «немой» кликабельный элемент — конвенция a11y, правило 2). `more_vert` — активен (см. AC-LS-19).
- **AC-LS-3** Ряд фильтр-чипов: «Все» (выбран, с галочкой), «Скоро», «Этот месяц», «Прошедшие». В MVP чипы
  **визуальны и неинтерактивны** (тап — no-op, «Все» постоянно выбран). Функциональная фильтрация — вне MVP ([09](09-edge-cases.md)).

## Грид upcoming
- **AC-LS-4** 2-колоночный грид карточек, высота ~204dp, асимметричные радиусы (циклически из набора
  `SHAPE_VARIANTS`, `m3-app.jsx:180-187`). Сортировка по возрастанию `targetDateTime` (ближайшее первым).
- **AC-LS-5** Карточка upcoming: иконка события (blob/rounded), чип-метка даты («09 МАЯ», локализованная) в
  правом верхнем углу, крупное число дней, строка «N дней осталось» (плюрализация), название (1 строка, ellipsis). Фон/текст — `container`/`onContainer` цвета события. Эталон `list-default.jpg`.
- **AC-LS-6** Extended FAB «Новое событие» (`primaryContainer`, иконка add, радиус 16dp) → открывает создание ([05](05-add-edit.md)).
- **AC-LS-7** Тап по карточке (upcoming или past) → полноэкранный счётчик соответствующего события ([06](06-navigation.md)).

## Свайп-удаление + undo
- **AC-LS-8** Свайп карточки влево показывает красный фон с иконкой delete и «Удалить» (эталон `list-swipe-delete.jpg`);
  завершённым считается свайп на **≥50% ширины** карточки (при отпускании <50% — карточка возвращается).
- **AC-LS-9** После удаления (свайпом или из past-счётчика) — snackbar «Удалено · Отменить»; «Отменить»
  восстанавливает событие. Реальное удаление из БД — после dismiss snackbar или истечения окна (soft-delete
  в памяти Store). Базовое окно — 5 с для зрячих пользователей; под активным screen reader окно
  продлевается через `calculateRecommendedTimeoutMillis` (AC-ACC-6/8) — действие Undo остаётся
  доступным. Commit происходит по истечении окна или свайпу снэкбара; «Отменить» — отменяет удаление.
  Работает и для upcoming, и для past карточек.
- **AC-LS-10** Если процесс убит до dismiss/таймаута — soft-delete отменяется, событие остаётся (источник
  истины — Room; Store держит только pending). Удаление alarm согласовано с soft-delete: alarm отменяется в
  момент soft-delete, при «Отменить» перепланируется на исходный `targetDateTime`, если он в будущем ([07](07-notifications.md) AC-NT-5).
- **AC-LS-10a** Если удаляется второе событие, пока snackbar первого ещё виден — первое удаляется немедленно
  (теряет окно undo), snackbar заменяется новым. *(Decision: один snackbar за раз; обоснование — простота против очереди undo.)*

## Секция «Прошедшие»
- **AC-LS-11** Ниже upcoming — заголовок-разделитель «Прошедшие · N» (иконка history) с кнопкой свернуть/развернуть
  («Свернуть ^»). Past-карточки — тем же гридом (вид — [03](03-past-events.md) AC-PE-4..5). Эталон `list-with-past.jpg`.
- **AC-LS-12** Состояние свёрнутости сохраняется между запусками (SettingsRepository, [01](01-data-model.md) AC-DM-10).

## Состояния списка
- **AC-LS-13** Loading: до получения данных — пустой/скелетный экран без ошибки.
- **AC-LS-14** Global empty (upcoming=0 И past=0): отдельный экран — blob-иконка hourglass_empty (teal),
  «Создайте первое событие», текст «Добавьте дату, которую ждёте — приложение посчитает остаток до секунды.»,
  крупная primary-кнопка «Новое событие». Сабтайтл шапки — «Пока ничего не запланировано». Эталон `empty-state.jpg`.
- **AC-LS-15** upcoming=0, но past>0: область upcoming показывает заглушку + кнопку «Новое событие»; ниже — обычная секция «Прошедшие». (НЕ global empty.)
- **AC-LS-16** Error (сбой загрузки): сообщение об ошибке вместо списка (не пустой экран, не краш).
- **AC-LS-17** FilteredEmpty (готовность к Post-MVP фильтрам): если фильтр даёт пусто, но события есть — текст «Нет событий по фильтру» БЕЗ кнопки «Новое событие».

## Настройки (вход)
- **AC-LS-19** `more_vert` в app bar содержит пункт «Настройки» (открывает выбор темы) и, при отсутствии разрешения на уведомления, пункт «Включить уведомления» ([07](07-notifications.md) AC-NT-12). Реализация настроек темы — диалог/bottom-sheet (3 режима), без отдельного экрана/модуля ([08](08-theme-localization.md) AC-TH-10).

## Доступность (экран) — по [конвенции](../../conventions/accessibility.md)
- **AC-LS-18** Тач-цели интерактивных элементов (чипы, карточки swipe-action, ячейки) ≥48×48dp (правило 3).
- **AC-LS-20** contentDescription: иконка события в карточке (название события), `more_vert` («Ещё»), FAB («Новое событие»), кнопка свайп-удаления, кнопка свернуть/развернуть секции. Декоративные элементы — `null`.
- **AC-LS-21** Snackbar «Удалено · Отменить» объявляется screen reader'у (liveRegion); действие «Отменить»
  доступно screen-reader-пользователю — окно продлевается через `calculateRecommendedTimeoutMillis`
  (базовое 5 с, под screen reader — до явного dismiss; AC-ACC-6/8, правило 6 конвенции). После успешного
  Undo screen reader получает отдельный анонс «Событие восстановлено».
- **AC-LS-22** Состояние выбора фильтр-чипа (визуального) и пустые/ошибочные состояния объявляются осмысленно; смена loading→content анонсируется.
